# 회의록 → 공유용 요약 메일 프로젝트

## 이 프로젝트의 목적

회의록(메모)을 입력하면, 팀에 공유할 수 있는 **요약 메일 초안**을 만드는 프로젝트입니다.
두 명의 에이전트가 협업합니다.

## 작업 흐름 (반드시 이 순서를 지킬 것)

사용자가 회의록 요약을 요청하면 아래 순서로 진행합니다.

1. **summary-writer** 에이전트를 호출하여 요약 메일 초안을 작성합니다.
2. 초안이 완성되면 **review-manager** 에이전트를 호출하여 초안을 검수시킵니다.
3. review-manager가 지적한 문제를 반영하여 **최종본**을 작성합니다.
4. 최종 답변은 아래 형식으로 출력합니다.

## 최종 출력 형식

```
## 📝 초안 (summary-writer)
(1단계 초안 전문)

## 🔍 검수 의견 (review-manager)
(2단계 지적 사항 전문)

## ✅ 최종본
(검수 반영 완성본)
```

## 공통 규칙

- 모든 출력은 한국어로 작성합니다.
- 회의록에 없는 내용을 지어내지 않습니다. 정보가 없으면 "확인 필요"라고 명시합니다.
- 요약 메일의 수신자는 회의에 참석하지 않은 팀원이라고 가정합니다.
