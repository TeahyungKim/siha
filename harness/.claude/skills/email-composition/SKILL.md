# 회의 요약 이메일 작성 스킬

필터링된 회의 내용을 수신자가 빠르게 파악할 수 있는 비즈니스 이메일로 변환한다.

## 이메일 구조 템플릿

```
제목: [회의 요약] {회의명} - {YYYY.MM.DD}

안녕하세요.
{날짜} 진행된 {회의명} 결과를 공유드립니다.

■ 회의 개요
- 일시: {날짜 및 시간}
- 참석: {역할 기반 참석자 목록}
- 목적: {회의 목적 한 줄}

■ 결정 사항
1. {결정 내용} — {결정 배경이 있다면 괄호 안에}
2. ...

■ Action Items
| 내용 | 담당 | 기한 |
|------|------|------|
| ... | ... | YYYY-MM-DD |

■ 후속 일정 (있는 경우)
- 다음 회의: {일시}

문의사항은 회신해 주시기 바랍니다.
감사합니다.
```

## 작성 원칙

**역피라미드 원칙**: 가장 중요한 결정사항을 먼저, 세부 내용은 뒤에 배치한다.

**Action Item 명확성**: 담당자·기한·내용이 한 줄에서 파악되어야 한다.
- 좋은 예: "숙소 예약 진행 | 운영담당 | 2026-06-30"
- 나쁜 예: "숙소 관련 처리"

**어조 일관성**:
- 결정사항: "~하기로 결정하였습니다", "~로 확정되었습니다"
- Action Item: "~를 진행해 주시기 바랍니다"

**잡담·여담 제외**: 회의 중 오간 비업무적 대화(식당 얘기, 과거 에피소드 등)는 절대 포함하지 않는다.

**정보 부재 처리**: 확인이 필요한 정보는 `[확인 필요: ...]` 플레이스홀더로 표시한다. 임의로 내용을 채우지 않는다.

**개인 정보 준수**: `_workspace/02_filtered_content.md`의 이름/역할 표기 방식을 그대로 따른다. 원본 분석 결과(`01_analysis.md`)의 원본 이름을 이메일에 직접 사용하지 않는다.
