# 회의록 분석 스킬

비정형 대화체 회의록을 읽고 후속 에이전트들이 활용할 구조화된 분석 결과를 생성한다.

## 분석 절차

1. **회의록 전체 읽기** — 요약 없이 원문 전체를 파악한다
2. **메타정보 추출** — 날짜·시간·장소/형태·회의 목적
3. **참석자 정보 추출** — 이름·직위·부서를 원문 그대로 기록 (필터링은 다음 단계)
4. **안건별 논의 분류** — 명시된 안건과 자연스럽게 흘러간 주제 모두 포함
5. **잡담·여담 식별** — 식당 얘기, 과거 에피소드, 개인 일정 등 비업무적 대화를 별도 표시
6. **결론 분류** — 아래 기준으로 상태 표시
7. **민감 정보 플래그** — 제거하지 않고 포함 여부만 체크리스트로 표시

## 결론 상태 분류 기준

| 상태 | 기준 |
|------|------|
| 결정됨 | 명확한 합의 표현 ("~하기로 결정", "~확정", "~승인") |
| 조건부 | 조건이 충족되면 실행 ("~이면 진행", "~결재 후 진행") |
| 보류 | 논의는 됐으나 명시적 결론 없음 |
| 미결 | 다음 회의 이월 또는 추가 정보 필요 |

## 대화체 회의록 처리 요령

회의록이 "김 팀장: ..." 형식의 대화체인 경우:
- 화자 발언을 순서대로 추적하여 안건의 흐름을 파악한다
- 담당자가 변경된 경우("한도윤 님이 어렵다면 박지호 님이 맡아주세요") → 최종 담당자만 기록
- 안건이 명시되지 않아도 주제 전환을 감지하여 안건을 재구성한다

## 출력 요건

- `_workspace/01_analysis.md`에 저장
- 원문의 표현을 가능한 유지 — 해석 추가 최소화
- 불명확한 내용은 `[불명확: ...]`로 표시
- 인사정보 포함 여부는 제거하지 않고 플래그만 표시
- 잡담·여담은 별도 섹션에 목록화 (이후 필터링 근거로 활용)
