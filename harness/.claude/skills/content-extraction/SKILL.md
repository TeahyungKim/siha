# 결정사항 & Action Item 추출 스킬

회의 분석 결과에서 이메일의 핵심 소재가 될 결정사항과 Action Item을 꼼꼼하게 추출한다.

## 추출 원칙

### 결정사항 추출
- **명시적 합의**: "~하기로 했다", "~로 결정되었다", "~확정"
- **암묵적 합의**: 반론 없이 수용된 제안, 투표 없이 채택된 방향
- **번복된 결정**: 회의 중 결정이 바뀐 경우 최종 결론만 기록, 번복 사실은 각주로
- **조건부 결정**: 조건과 함께 기록 ("B 후보 연봉 협의 성공 시 채용 확정" 등)

### Action Item 추출
동사 기반으로 탐색한다:
- 강한 표현: "~해주세요", "~하겠습니다", "~담당해 주시기 바랍니다"
- 약한 표현도 포착: "~를 검토하겠습니다", "~쪽에서 확인해 보겠습니다"
- 후속 회의/보고 일정 관련 사항도 포함

### 담당자/기한 처리
- 담당자 불명확 시 `[담당자 미정]` — 임의 배정 금지
- 기한 없는 항목 `[기한 없음]` — 임의 설정 금지
- 기대 산출물(무엇을 만들어야 하는지)을 명시
- 담당자가 변경된 경우 최종 담당자로 기록

## 우선순위 기준

| 우선순위 | 기준 |
|---------|------|
| 상 | 즉각 실행 필요 또는 다른 작업의 블로커 |
| 중 | 정해진 기한 내 완료 필요 |
| 하 | 기한 없음 또는 장기 과제 |

## 누락 방지 체크

분석 결과의 안건 목록을 순서대로 검토하면서:
1. 안건마다 결정사항이 있었는지 확인
2. 안건마다 Action Item이 발생했는지 확인
3. 미결 사항도 "확인 필요" Action Item으로 등록
4. 담당자 미정 항목을 마지막에 한번 더 검토
