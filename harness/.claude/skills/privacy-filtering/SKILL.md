# 개인정보 보호 필터링 스킬

이메일로 배포될 회의 내용에서 인사정보 및 개인 민감 정보를 제거하여 정보 보호를 보장한다.

## 필터링 대상

### 무조건 제거
- 급여·연봉·인센티브·성과급 등 보상 관련 수치와 논의
- 개인별 성과 평가 내용·등급·점수
- 채용 지원자 면접 평가 점수·순위·상세 평가 내용
- 채용 지원자 희망 연봉 및 연봉 협상 내용
- 평판 조회 내용 (긍정·부정 불문)
- 징계·경고·PIP(성과개선계획)·해고·권고사직 관련 모든 내용
- **확정 전** 인사발령 정보 (공식 발표 전)
- 건강·질병·가족 상황 등 사생활 관련 정보

### 맥락에 따라 처리
- **참석자 이름**: 역할로 대체 가능 ("김OO 팀장" → "팀장")
  - Action Item 담당자 추적에 필요하면 역할명으로 유지
- **채용 지원자 이름**: "1순위 지원자", "과제 전형 대상자" 등으로 대체
- **조직 개편**: 공식 발표된 것이면 유지, 미확정이면 "[조직 관련 사항은 추후 공식 안내]"로 대체
- **잡담·여담**: meeting-analyzer가 식별한 비업무적 대화는 이메일 내용에서 제외

### 유지 가능
- 역할/직위 기반 담당자 표기 (Action Item 추적 필수)
- 공식 발표된 인사 정보
- 채용 절차 진행 결과의 방향성 (상세 이유 없이: "오퍼 진행 예정", "추가 전형 예정")

## 처리 원칙

**보수적 원칙**: 포함 여부가 모호하면 제거한다. 필요 시 담당자에게 확인을 권고한다.

**가독성 유지**: 제거 후 문맥이 어색해지면 자연스러운 대체 표현을 삽입한다.

**로그 보존**: 원본 내용 전체가 아닌 "제거 사유와 처리 방식"만 기록한다.

## 출력

- `_workspace/02_filtered_content.md`: 필터링 완료된 핵심 내용
- `_workspace/02_privacy_log.md`: 제거 로그 (감사 추적용)

### 로그 형식

```markdown
# 개인정보 필터링 로그

## 제거된 항목
| # | 내용 요약 | 제거 사유 | 처리 방식 |
|---|---------|---------|---------|
| 1 | 지원자 희망 연봉 수치 | 보상 정보 | 삭제 |
| 2 | 평판 조회 내용 | 채용 민감 정보 | 삭제 |

## 주의 필요 항목 (제거 안 했으나 검토 권고)
- ...
```
