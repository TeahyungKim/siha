# Markdown 핵심 문법 실습 정리 (비개발자용)

> 목적: Claude Code 실습에서 `CLAUDE.md`, `SKILL.md` 등을 직접 읽고 쓸 수 있게 하는 최소한의 Markdown 익히기

---

## 0. Markdown이 뭐고, 왜 배우나 

- Markdown은 **일반 텍스트에 간단한 기호를 붙여 서식을 표현하는 표기법**입니다.
- 워드처럼 버튼을 누르는 게 아니라, `#`  `*`  `-` 같은 기호를 직접 타이핑해서 사용합니다.
-  **왜 우리에게 필요한가**: Claude Code의 핵심 설정 파일들이 전부 Markdown입니다.
-  `CLAUDE.md` — 에이전트에게 주는 작업 지침서
-  `SKILL.md` — 슬래시 커맨드로 호출하는 스킬 정의
- 에이전트 정의 파일 (`.claude/agents/*.md`)
- 비유: **Markdown은 "에이전트가 읽는 업무 매뉴얼의 양식"**입니다. 양식을 알아야 매뉴얼을 쓸 수 있습니다.

---
## 1. 제목 (Heading) — 문서의 뼈대

`#` 의 개수 = 제목의 크기 단계. `#` 뒤에 **공백 한 칸** 필수.

```markdown
# 가장 큰 제목 (문서 제목)
## 두 번째 단계 (큰 섹션)
### 세 번째 단계 (소제목)
```

> 실무 연결: `CLAUDE.md`에서 `## 역할`, `## 작업 규칙` 같은 제목으로 지침을 구조화하면 에이전트가 훨씬 잘 따르게 됩니다.

---

  

## 2. 강조 — 굵게, 기울임

| 쓰고 싶은 것 | 입력 | 결과 |
|---|---|---|
| 굵게 | `**중요**` | **중요** |
| 기울임 | `*참고*` | *참고* |
| 굵은 기울임 | `***매우 중요***` | ***매우 중요*** |
| 취소선 | `~~삭제됨~~` | ~~삭제됨~~ |



> 실무 연결: 만일 중요한 단어나 문장이라면 강조 표현을 하면 마찬가지로 에이전트가 훨씬 잘 따르게 됩니다.

---
  

## 3. 목록 — 가장 많이 쓰게 될 문법  

**순서 없는 목록**: `-` 또는 `*` 뒤에 공백  

```markdown
- 회의록 요약
- 액션 아이템 추출
- 담당자 지정
```


**순서 있는 목록**: 숫자 + 마침표 + 공백

```markdown
1. 원문을 읽는다
2. 핵심을 추린다
3. 검토자에게 넘긴다
```


**들여쓰기로 하위 항목** (공백 2~4칸):
 
 ```markdown
- 1차 검토
- 맞춤법
- 용어 일관성
- 2차 검토
```

  

> 실무 연결: 에이전트에게 작업 절차를 줄 때 순서 있는 목록이 가장 효과적입니다.

---
  

## 4. 체크리스트 (Task List)
  
```markdown
- [ ] 초안 작성
- [x] 자료 조사
- [ ] 임원 보고
```


-  `[ ]` = 미완료, `[x]` = 완료. **대괄호 안 공백/x 위치가 정확해야** 렌더링됩니다.

> 실무 연결: 에이전트에게 "결과를 체크리스트로 정리해줘"라고 하면 이 형식으로 돌려줍니다. 읽는 법을 알아야 합니다.

---

## 5. 인용 (Blockquote) — 원문과 의견 구분

```markdown
> 고객 리뷰: "예약 취소가 너무 복잡해요."

위 리뷰에 대한 분석: 취소 절차가 3단계 이상이다.
```


`>` 한 칸 뒤에 내용. 인용 안에 인용도 가능합니다. (`>>`).

> 실무 연결: 고객 VOC 원문 인용 + 분석을 구분할 때, 에이전트 지침에서 예시 문구를 표시할 때 유용합니다.

---


## 6. 코드 표기 — "건드리지 말고 그대로" 표시  

비개발자에게도 필요한 이유: **버튼명, 명령어, 파일명, 그대로 보여줄 문구**를 표시할 때 씁니다.
**인라인 코드**: 백틱(`` ` ``) 1개로 감싸기 — 키보드에서 보통 숫자 1 왼쪽, 물결(~)과 같은 키  

```markdown
설정에서 `permissions` 항목을 확인하세요.
```


**코드 블록**: 백틱 3개로 위아래 감싸기
````markdown
```
/init
/compact 핵심만 남겨줘
```
````

> 실무 연결: 에이전트 지침에서 "출력 형식 예시"를 보여줄 때 코드 블록으로 감싸면, 에이전트가 그것을 지시문이 아니라 **견본**으로 인식합니다.

## 7. 표 (Table)

```markdown
| 항목 | 담당 | 마감 |
|---|---|---|
| 랜딩 페이지 카피 | 지민 | 6/15 |
| 릴리스 노트 | 수현 | 6/20 |
```


- 1행: 제목 줄 / 2행: 구분선(`|---|` 반복) / 3행부터: 내용
- ⚠️ **2행 구분선을 빼먹으면 표가 안 됩니다.** 가장 흔한 표 실수.
- 칸 너비를 맞춰 예쁘게 쓸 필요 없다. 파이프(`|`) 개수만 맞으면 됩니다.

---
  

## 8. 구분선과 줄바꿈 — 의외로 자주 막히는 부분

**구분선**: 빈 줄 사이에 `---` 단독 한 줄

 

**줄바꿈 규칙** (⚠️ 비개발자가 가장 많이 당황하는 지점):

- Enter 한 번만 치면 → 미리보기에서 **한 줄로 이어져 보일 수도 있습니다. (아닐 수도 있습니다)**
- 만일 같은 문단 안에서 줄 바꿈이 안된다면 → 줄 끝에 **공백 2칸** 후 Enter
- 문단을 나누려면 → **빈 줄 하나를 사이에** 둡니다 (Enter 두 번)
